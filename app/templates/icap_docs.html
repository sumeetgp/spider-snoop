Configuration Examples

Squid Proxy

squid.conf

Configure Squid to offload content to SpiderCob for DLP scanning. 

# Enable ICAP
icap_enable on
icap_preview_enable on
icap_preview_size 1024
icap_service_failure_limit -1
icap_send_client_ip on

# Define SpiderCob Service (Secure ICAP)
icap_service service_req reqmod_precache bypass=0 icaps://icap.spidercob.com:443/reqmod
icap_service service_resp respmod_precache bypass=0 icaps://icap.spidercob.com:443/respmod

# Authentication (Add Header)
# Note: Replace jwt_token with your long-lived token
request_header_add X-ICAP-Auth "Bearer <your_jwt_token>" all

# Apply to Traffic
adaptation_access service_req allow all
adaptation_access service_resp allow all

Symantec / BlueCoat ProxySG

CPL / VPM 
// Visual Policy Manager Configuration
1. Create a "Response Analysis" object.
2. Select "ICAP" as the provider.
3. Service URL: icaps://icap.spidercob.com:443/respmod
// Note: Ensure "Secure ICAP" is enabled in ProxySG settings

// CPL (Policy)
<Content-Security-Policy>
    icap_server(spider_cob)
    
// Header Injection for Auth
<Proxy>
    brand.add_auth_header(yes)

define brand add_auth_header
    set(request.x_header.X-ICAP-Auth, "Bearer <your_jwt_token>")
end

Security & Token Lifecycle

The ICAP server requires authentication for scanning connections (REQMOD/RESPMOD). OPTIONS requests for health checks are allowed without authentication. For scanning, you must pass a valid JWT in the Authorization or X-ICAP-Auth header.

  • Tokens generated via login expire in 60 minutes.
  • For permanent infrastructure links (like Squid/Bluecoat), verify you have a Service Account or use the API to generate long-lived credentials (Coming Soon).
